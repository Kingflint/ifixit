# QA checklist

Before each release, walk the following flows:

- [ ] Sign up with email + password; verify mailbox link.
- [ ] Sign in with Google; confirm profile created.
- [ ] Book a repair as a signed-out user (sign-up gate triggers).
- [ ] Book a repair as a signed-in user; tracking code visible.
- [ ] Track an order via the public tracker.
- [ ] Open the admin dashboard as the seeded admin.
- [ ] Toggle store mode; confirm catalogue renders.
- [ ] Buy a product end-to-end with Paystack test mode.
- [ ] Cancel an in-flight repair; confirm status badge.
